    Replace `main` with a fresh Laravel 13 + `laravel/ai` scaffold using an orphan branch. Implements D-02 (orphan reset), D-04 (no migration of legacy files — they only live on `legacy/v1`).

    PRESERVATION FIRST — before deleting anything, copy these out of the working tree to a temp location so they survive the wipe:
    - `.planning/` directory (entire tree — contains all phase artifacts including this PLAN.md)
    - `CLAUDE.md` (project instructions)
    - `SKD.md` (Laravel AI SDK reference docs)
    - `.env` (only if it contains real provider API keys you want to keep — otherwise skip)

    Concrete steps (run from project root `/home/rec/Documents/Faculdade/pc3-submission-evaluator`):

    1. Create temp preservation dir:
       ```
       TMP=$(mktemp -d)
       cp -r .planning "$TMP/"
       cp CLAUDE.md "$TMP/"
       cp SKD.md "$TMP/"
       [ -f .env ] &amp;&amp; cp .env "$TMP/.env.preserved" || true
       echo "Preserved to: $TMP"
       ```

    2. Switch to a new orphan branch:
       ```
       git checkout --orphan new-main
       ```

    3. Wipe the working tree (clears index of all paths, removes files):
       ```
       git rm -rf --cached . 2>/dev/null || true
       git clean -fdx -e .planning -e CLAUDE.md -e SKD.md
       ```
       Note: `-e` excludes the planning/docs files we want to keep. `-fdx` removes untracked files (including `vendor/`, `node_modules/`).

    4. Remove remaining tracked files in working tree (the `git clean` above only removes untracked):
       ```
       find . -mindepth 1 -maxdepth 1 \( -name .git -o -name .planning -o -name CLAUDE.md -o -name SKD.md \) -prune -o -exec rm -rf {} +
       ```

    5. Install Laravel 13 into the (now mostly empty) directory. Use a temp subdirectory because `composer create-project` won't run into a non-empty target:
       ```
       composer create-project "laravel/laravel:^13.0" /tmp/fresh-laravel-13
       cp -r /tmp/fresh-laravel-13/. .
       rm -rf /tmp/fresh-laravel-13
       ```
       This puts every file from the Laravel 13 skeleton into the project root, preserving `.planning/`, `CLAUDE.md`, and `SKD.md` already present.

    6. Restore preserved planning files (in case any were overwritten — they shouldn't be, but be defensive):
       ```
       cp -r "$TMP/.planning" .
       cp "$TMP/CLAUDE.md" .
       cp "$TMP/SKD.md" .
       ```

    7. Configure SQLite as the DB:
       - Edit `.env`: set `DB_CONNECTION=sqlite` (Laravel 13 defaults to sqlite already, verify the line exists and is uncommented).
       - Remove or comment out `DB_HOST`, `DB_PORT`, `DB_DATABASE`, `DB_USERNAME`, `DB_PASSWORD` lines (they aren't needed for sqlite).
       - Same edits in `.env.example`.
       - Create the SQLite file: `touch database/database.sqlite`.

    8. Install `laravel/ai`:
       ```
       composer require "laravel/ai:^0.6"
       php artisan vendor:publish --provider="Laravel\\Ai\\AiServiceProvider"
       ```
       Verify `config/ai.php` was created.

    9. Run migrations to verify the install:
       ```
       php artisan migrate --no-interaction
       ```
       This creates Laravel's default tables PLUS the AI SDK's `agent_conversations` and `agent_conversation_messages` tables. Per Pitfall 2 in RESEARCH.md, these are expected and harmless.

    10. Stage and commit on `new-main`:
        ```
        git add -A
        git commit -m "chore: fresh Laravel 13 scaffold with laravel/ai SDK"
        ```

    11. Replace `main` with `new-main`:
        ```
        git branch -D main 2>/dev/null || true
        git branch -m new-main main
        ```

    12. Force-push to origin (D-03 — solo developer, no parallel contributors):
        ```
        git push origin main --force
        ```

    13. Set `main` as the current branch tip and clean up: `rm -rf "$TMP"`.

    Do NOT touch `legacy/v1` during this task — it must remain untouched.

    If any step fails, STOP and report the error; do not press forward with a half-complete reset.
