    First, confirm the FQN of `StructuredAgentResponse`. Run:

    Exact file contents (assuming `Laravel\Ai\Responses\StructuredAgentResponse` is correct — adjust the `use` line if the grep above shows a different namespace):
    ```php
    <?php

    declare(strict_types=1);

    namespace App\DTOs;

    use Laravel\Ai\Responses\StructuredAgentResponse;

    final class ProviderResult
    {
        private function __construct(
            public readonly string      $provider,
            public readonly string      $model,
            public readonly string      $diagnosis,
            public readonly Pc3Category $pc3Category,
            public readonly string      $feedback,
            public readonly float       $confidence,
            public readonly int         $tokensInput,
            public readonly int         $tokensOutput,
            public readonly string      $requestId,
            public readonly string      $promptVersion,
        ) {}

        public static function fromPrismResponse(
            StructuredAgentResponse $response,
            string $provider,
            string $model,
            string $requestId,
            string $promptVersion,
        ): self {
            return new self(
                provider:      $provider,
                model:         $model,
                diagnosis:     (string) $response['diagnosis'],
                pc3Category:   Pc3Category::from((string) $response['pc3_category']),
                feedback:      (string) $response['feedback'],
                confidence:    max(0.0, min(1.0, (float) $response['confidence'])),
                tokensInput:   (int) $response['tokens_input'],
                tokensOutput:  (int) $response['tokens_output'],
                requestId:     $requestId,
                promptVersion: $promptVersion,
            );
        }
    }
    ```

    Notes on the construction approach:
    - `private function __construct(...)` blocks `new ProviderResult(...)` from outside the class — D-07's "single construction path" is enforced by language.
    - The class is `final` (cannot be subclassed to expose constructor) — combined with `readonly` properties, the DTO is fully immutable post-construction.
    - We do NOT use `final readonly class` syntax (PHP 8.2+) directly because property-level `readonly` is more explicit and equally enforced.
